Architecture Approach

How should we structure the macOS menu bar app?

A

SwiftUI + SwiftData (Recommended)

Modern, minimal code approach

  • MenuBarExtra for the menu bar icon + popover
  • SwiftData (Apple's modern ORM) for local persistence
  • Single-window popover with tab navigation
  • Requires macOS 14+ (Sonoma)

Pros

  • Least code — SwiftData handles storage automatically
  • Declarative UI with SwiftUI
  • Easy to extend later (iCloud sync is one toggle)

Cons

  • Requires macOS 14+
  • SwiftData is newer, fewer community resources
B

SwiftUI + JSON File Storage

Simple, no framework dependencies

  • MenuBarExtra for the menu bar icon + popover
  • Plain JSON files in Application Support directory
  • Manual Codable encoding/decoding
  • Works on macOS 13+

Pros

  • No ORM dependency — full control over data
  • Easy to inspect/debug (just read the JSON)
  • Supports older macOS versions

Cons

  • More boilerplate for CRUD operations
  • Manual migration if data model changes
C

SwiftUI + CoreData

Battle-tested but verbose

  • MenuBarExtra for the menu bar icon + popover
  • CoreData with NSManagedObject models
  • Works on macOS 12+

Pros

  • Very mature, tons of resources
  • Handles complex queries well

Cons

  • Verbose — lots of boilerplate
  • Overkill for this app's data complexity
  • Apple is moving toward SwiftData
